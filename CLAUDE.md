# OpenLieroX Wiki — Jekyll re-creation

A static **Jekyll** site that faithfully imitates the original
**OpenLieroX MediaWiki** (`http://www.openlierox.net/wiki/`), reconstructed
from the **Internet Archive Wayback Machine**. The original wiki is long
gone; this project rebuilds it as a static site whose page content is stored
as **Markdown**.

Deployed via GitHub Actions to GitHub Pages:
**https://klirktag.github.io/openlierox-wiki-jekyll/**

## How it works

The original wiki ran MediaWiki with a custom skin called `openlierox`
(a MonoBook variant built on the WordPress "MistyLook" theme). This project
reproduces that skin as a Jekyll layout + CSS, and stores every article as a
Markdown document in a Jekyll collection.

### Project layout

| Path | Purpose |
|------|---------|
| `_config.yml` | Jekyll config. Defines the `wiki` collection (`permalink: /wiki/index.php/:path/`). |
| `_layouts/wiki.html` | The page template — a faithful copy of the original `openlierox` MediaWiki skin (top banner, green nav bar, sidebar boxes, footer). |
| `assets/css/style.css` | The original skin's `main.css`, with asset `url()` paths rewritten to `../skin/`. Custom additions are at the bottom. |
| `assets/skin/` | Skin images downloaded from the Wayback Machine (logo, backgrounds, bullets, icons). |
| `assets/js/wiki.js` | Client-side search + "Random page" (the original used MediaWiki server-side). |
| `_wiki/*.md` | One Markdown file per wiki article. Generated by `scripts/fetch_wiki.py`. |
| `_wiki/Special_*.html` | Static stand-ins for MediaWiki special pages (Search, Random, All pages). |
| `_plugins/baseurl_links.rb` | Build hook: prefixes root-relative `/wiki/` and `/assets/` links with `site.baseurl` so the project page works under `/openlierox-wiki-jekyll/`. |
| `_plugins/category_pages.rb` | Generator: builds a `Category:<Name>` page for every category used in front matter. |
| `scripts/fetch_wiki.py` | Scraper/converter: pulls archived pages from the Wayback Machine and writes the Markdown in `_wiki/`. |
| `scripts/to_mediawiki.py` | Converts the Markdown articles back to MediaWiki wikitext + a Special:Import XML dump, written into `mediawiki/`. |
| `mediawiki/` | MediaWiki export for migrating the wiki to a real MediaWiki install elsewhere. One `.wiki` file per article plus `openlierox-wiki.xml`. See [mediawiki/README.md](mediawiki/README.md). |
| `index.html` | Root redirect to the Main Page (the original wiki opened there). |
| `.github/workflows/jekyll.yml` | Builds with Jekyll and deploys to GitHub Pages. |

### URL scheme

The original served articles at `/wiki/index.php/<Page_Name>`. We mirror that
with the collection permalink `/wiki/index.php/:path/`, e.g.
`/wiki/index.php/Liero_1.0/`. Page/file names use underscores for spaces and
replace `:` and `/` with `_` (e.g. `Lua:worm` → file `Lua_worm.md`).

### Content conversion (`scripts/fetch_wiki.py`)

1. Queries the Wayback CDX API for every archived `index.php/*` page.
2. For each page it keeps *all* snapshots, ranked so healthy captures
   (≈2009–2014, before the wiki started returning PHP errors) are tried
   first, newest-first within a tier.
3. Downloads the raw capture (`id_` modifier — no Wayback toolbar), validates
   it isn't a PHP error / empty page, and falls back to an older snapshot if
   it is.
4. Strips the MediaWiki chrome, rewrites internal links to the new URL
   scheme, converts the body to Markdown with `markdownify`, and keeps tables
   and image thumbnails as small inline HTML blocks.
5. Writes `_wiki/<name>.md` with YAML front matter (`title`, `categories`,
   `last_modified`, `archived_url`) and records referenced images in
   `scripts/images.tsv`.

The script is **resumable** — it skips pages already present in `_wiki/`.
`_wiki/Main_Page.md` is hand-crafted (the styled welcome boxes) and is left
untouched by the scraper.

Re-run with: `python3 scripts/fetch_wiki.py`
(needs `pip install --user --break-system-packages markdownify beautifulsoup4`)

### Migrating to a real MediaWiki

`scripts/to_mediawiki.py` reads `_wiki/*.md` and writes:

- `mediawiki/<Name>.wiki` — one wikitext file per page
- `mediawiki/openlierox-wiki.xml` — bulk Special:Import / importDump.php dump

Re-run with: `python3 scripts/to_mediawiki.py`. See
[mediawiki/README.md](mediawiki/README.md) for upload instructions.

## Local development

Ruby/Jekyll are not required on the host — use Docker:

```sh
docker run --rm -v "$PWD":/site -w /site -p 4000:4000 ruby:3.3 \
  bash -c "bundle install && bundle exec jekyll serve --host 0.0.0.0 --baseurl ''"
```

Then open <http://localhost:4000/>. (`--baseurl ''` for local; production uses
`/openlierox-wiki-jekyll`.)

## Deployment

Pushing to `master` triggers `.github/workflows/jekyll.yml`, which builds with
Jekyll (custom `_plugins/` are loaded because we build ourselves rather than
via the classic `github-pages` gem) and deploys to GitHub Pages.

**One-time setup:** repo *Settings → Pages → Source → "GitHub Actions"*.

## Notes / conventions

- Content lives in Markdown; tables and floated image thumbnails are kept as
  inline HTML because Markdown cannot express them faithfully.
- Links inside Markdown are written clean and root-relative
  (`/wiki/index.php/Foo/`); `_plugins/baseurl_links.rb` adds the base path at
  build time. Links in layouts use the `relative_url` filter directly.
- "Red links" (articles that never existed on the original wiki) are rendered
  as plain text.
- This is an archival re-creation: a small "Re-created from the Wayback
  Machine" note sits in the footer; the article bodies are kept authentic.
